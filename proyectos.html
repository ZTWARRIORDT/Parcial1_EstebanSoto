Cómo Añadir un Archivo .gitignore

Un archivo `.gitignore` es crucial para gestionar qué archivos y directorios deben ser ignorados por Git. Esto es especialmente útil para evitar que archivos generados automáticamente, archivos de configuración local, o información sensible se incluyan en el repositorio. A continuación, te explicamos cómo crear y configurar un archivo `.gitignore`:

  1. Crear el Archivo: En la raíz de tu proyecto, crea un archivo llamado .gitignore. Puedes hacerlo con un editor de texto o usando el siguiente comando en la terminal:
    2. touch .gitignore
  2. Agregar Registros al Archivo: Abre el archivo `.gitignore` en tu editor de texto y agrega las rutas de los archivos y carpetas que quieres ignorar. Aquí algunos ejemplos comunes:
    3. # Ignorar directorio de dependencias
node_modules/

# Ignorar archivos de logs
*.log

# Ignorar archivos de sistema de macOS
.DS_Store
  3. Guardar y Añadir al Repositorio: Guarda el archivo y asegúrate de añadirlo a tu repositorio Git. Usa los siguientes comandos:
    4. git add .gitignore
git commit -m "Añadido archivo .gitignore"
  4. Verificar el Estado: Para asegurarte de que los archivos especificados en `.gitignore` están siendo ignorados correctamente, puedes usar el comando:
    5. git status

    Si todo está configurado correctamente, los archivos y directorios listados en el archivo `.gitignore` no aparecerán en el estado del repositorio.
